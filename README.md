# SaltCity Leadership PWA

Mobile-first pastoral operations companion for SaltCity Church leadership.
Built with Next.js 14 (App Router), TypeScript, Tailwind CSS, and Supabase.

Product context lives in `CLAUDE.md`; the full build brief in `HANDOFF.md`;
database schema in `SCHEMA.md`; the approved pixel reference in `SaltCity Final.html`.

## Setup

### 1. Install

```bash
npm install
```

### 2. Supabase

Create a project at [supabase.com](https://supabase.com), then either:

**Via the CLI** (recommended):

```bash
npx supabase login
npx supabase link --project-ref YOUR_PROJECT_REF
npx supabase db push          # applies supabase/migrations/00000000000001_init.sql
```

**Or via the dashboard:** paste the contents of
`supabase/migrations/00000000000001_init.sql` into the SQL editor and run it.

Optionally run `supabase/seed.sql` for development data (create at least one
auth user first — the signup trigger creates their profile row automatically;
then set their `role` / `company_id` on the `profiles` table).

### 3. Environment

Replace the placeholder values in `.env.local` with your project's credentials
(Supabase dashboard → Settings → API). See `.env.local.example`.

### 4. Run

```bash
npm run dev        # development (service worker disabled)
npm run build      # production build (generates public/sw.js)
npm run start      # serve the production build
npm run typecheck  # tsc --noEmit
npm run lint       # eslint
```

## Users & roles

Auth users sign up via Supabase (no in-app signup). A database trigger creates
a `profiles` row on signup; an admin then sets the role:

| Role | Access |
|---|---|
| `company_leader` / `assistant_leader` | Their company only: weekly report, assigned follow-ups, own tasks |
| `church_admin` | Everything, including report review/flag and publishing |
| `church_office` | Member & follow-up management; cannot review/flag reports |

Row Level Security enforces all of this at the database level.

## Notes

- **PWA icons** in `public/icons/` are solid-berry placeholders generated by
  `scripts/generate-icons.mjs` — replace with branded icons (logo on `#6B2540`)
  before launch.
- `src/lib/database.types.ts` is hand-written to match the migration. After any
  schema change, regenerate it:
  `npx supabase gen types typescript --project-id YOUR_PROJECT_REF > src/lib/database.types.ts`
- `@supabase/supabase-js` is pinned to `2.45.4` — `@supabase/ssr@0.5.2` depends
  on that package layout. Upgrade both together if you bump either.
